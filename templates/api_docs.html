Café & Wi-Fi API

A RESTful API serving data on Jacksonville, FL's best cafés for remote working. Query café listings, discover random spots, add new venues, update prices, and report closures.

Base URL: http://localhost:5001/api Format: JSON Auth: None (open)

Quick Start

The API is open and requires no authentication. All responses are in JSON format. To get started, fetch the full list of cafés:

GET /api/cafes 
curl http://localhost:5001/api/cafes

Endpoints

GET /api/cafes

Returns all cafés, ordered alphabetically. Optionally filter by neighborhood.

Query Parameters

ParamTypeRequiredDescription
locationstringNoFilter by neighborhood (case-insensitive). e.g. Shoreditch

GET /api/random

Returns a single randomly selected café. Perfect for a "Feeling Lucky" feature.

Example Response

{
  "cafe": {
    "id": 7,
    "name": "Solune Coffee",
    "location": "Riverside",
    "has_wifi": true,
    ...
  }
}
POST /api/add

Adds a new café to the database. Expects a JSON body.

Request Body (JSON)

FieldTypeRequiredDescription
namestringYesCafé name (must be unique)
map_urlstringYesGoogle Maps URL
img_urlstringYesImage URL for the café
locationstringYesNeighborhood name
has_wifibooleanNoDefault: false
has_socketsbooleanNoDefault: false
has_toiletbooleanNoDefault: false
can_take_callsbooleanNoDefault: false
seatsstringNoe.g. "20-30"
coffee_pricestringNoe.g. "$2.80"

Example Request

curl -X POST http://localhost:5001/api/add \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Sago Coffee",
    "map_url": "https://maps.google.com/...",
    "img_url": "https://example.com/sago_coffee.jpg",
    "location": "Jacksonville Beach",
    "has_wifi": true,
    "has_sockets": true,
    "has_toilet": true,
    "can_take_calls": true,
    "seats": "10-20",
    "coffee_price": "$3.50"
  }'

Success Response 201

{
  "response": { "success": "'The Roastery' was added successfully." },
  "cafe": { ... }
}

Error Responses

StatusReason
400Missing required fields
409Café name already exists
PATCH /api/update-price/<cafe_id>

Updates the coffee price for a specific café.

Query Parameters

ParamTypeRequiredDescription
new_pricestringYesThe updated price, e.g. $3.00

Example

curl -X PATCH "http://localhost:5001/api/update-price/1?new_price=$3.00"
DELETE /api/delete/<cafe_id>

Removes a café from the database by its ID.

Example

curl -X DELETE http://localhost:5001/api/delete/5
DELETE /api/report-closed/<cafe_id>

Reports a café as permanently closed. Semantically different from /api/delete — this represents a user reporting that a real-world venue has shut down.

Example

curl -X DELETE http://localhost:5001/api/report-closed/12

Error Format

All errors follow a consistent structure:

{
  "error": {
    "Not Found": "No café found with id 99."
  }
}
StatusMeaning
200Success
201Created
400Bad request / missing params
404Resource not found
409Conflict (duplicate)

Running Locally

# 1. Clone the repository
git clone https://github.com/your-username/cafe-wifi.git
cd cafe-wifi

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run the app
python app.py

# The app will be available at http://localhost:5001
# The API at http://localhost:5001/api/cafes
# These docs at http://localhost:5001/api/docs

Note: The database resets to its original state on every restart, so feel free to experiment with adding, updating, and deleting cafés.
